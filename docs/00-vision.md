# 00 — Vision

## Purpose

This document explains why CasaStudio exists, what problem it solves, and what kind of product it should become.

## Problem

During a real home renovation, many design decisions must be made before the work is finished. Once walls are plastered, floors are installed, or technical preparations are closed, some changes become expensive, disruptive, or impossible.

Traditional workflows often require switching between multiple tools: CAD, 3D modelling software, rendering software, AI image generator, and manual file organization.

## Product idea

CasaStudio provides a single workflow:

1. describe the home as structured data;
2. visualize the floor plan in 2D;
3. generate a navigable 3D scene;
4. save meaningful viewpoints;
5. export a viewpoint as a clean reference image;
6. use an AI image generation provider to create interior design proposals;
7. organize generated renders in a gallery / decision timeline.

## Product identity

CasaStudio is not only a drawing tool and not only an AI wrapper.

It is an AI-assisted design environment where the project data describes the house, the 2D and 3D viewers represent that data, the AI generates visual proposals from accurate geometry references, and the gallery helps compare alternatives and make decisions.

## What CasaStudio is not

For the MVP, CasaStudio is not a complete CAD system, a BIM platform, Blender, SketchUp, a photorealistic render engine, a generic ChatGPT wrapper, or an app that generates consistent furnished 3D scenes from AI.

## Guiding principle

> The house is the data. Everything else is a representation of that data.

The project model must be treated as the single source of truth.

## Long-term vision

After the MVP, CasaStudio may evolve into a broader platform with a richer 2D editor, more advanced 3D modelling, material and lighting management, AI design presets, render comparison tools, cloud projects, collaboration, and AI-generated structured design proposals.
